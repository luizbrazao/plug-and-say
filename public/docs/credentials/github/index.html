GitHub Credential (PlugandSay)

Guide for setting up GitHub credentials for issue and PR automation.

When to use

Use this credential when your agents need to create GitHub Issues and Pull Requests.

Authentication methods

1. API Access Token (Classic)

Recommended for initial setup and testing.

2. OAuth2

Planned advanced flow for centralized governance and token rotation.

Required fields

  • GitHub Server (default: https://api.github.com)
  • User (optional)
  • Access Token (required)
  • Default Repository (optional, format owner/repo)

Create a Personal Access Token (Classic)

  1. Go to GitHub Settings → Developer settings.
  2. Open Personal access tokens → Tokens (classic).
  3. Generate a new token with minimum required scopes.

Suggested scopes: repo, read:org (if needed), and workflow (if needed).

Troubleshooting

  • 401 Unauthorized: token invalid or expired.
  • 404 Not found: repository path incorrect or missing permission.
  • 403 Forbidden: insufficient scope or API limit reached.

Tools mapped to this credential

  • create_github_issue
  • create_pull_request
